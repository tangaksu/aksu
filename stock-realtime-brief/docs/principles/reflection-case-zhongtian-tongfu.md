## 维度 1：决策本身的问题（不留情面）

### 🚨 错误 1：**误用"主线锁定原则"**

```
金融博士铁律 1 说: "牛市资金集中在政策×资金×景气度三重共振的主赛道"
铁律的真实含义: 锁定主线 = 不要去碰 没有资金的冷门股
铁律没说的: ≠ "已经在板块龙头的浮盈仓位 应该卖出"

我的误用:
  ❌ "中天是支线 → 应该减仓"
  正确逻辑:
  ✅ "中天是支线 → 不应该新建仓 / 不应该重仓"
  
中天 1300 股 ≠ 新建仓 / ≠ 重仓
中天 1300 股 = 已经持有 + 浮盈 +40% 的小仓位
完全不在 "主线锁定铁律" 的 应用范围内
```

### 🚨 错误 2：**忽视 同板块同涨跌风险**

```
我推荐: 卖中天（海缆光纤）→ 买通富（CoWoS）

通富 + 罗博 + 华工 + 源杰 同涨同跌相关性 极高

风险结构:
  中天 (海缆) = 与 CPO/CoWoS 弱相关 → 分散
  通富 (CoWoS) = 与 罗博/华工 强相关 → 集中
  
换仓结果:
  减少了 分散度
  增加了 集中度
  这是 5/29 单日 "通富 + 华工 + 罗博" 一起跌 的 根本原因

```

```
  main_line_tracker 输出 主线/支线分类

  • 这个分类 适用于 新建仓 决策
  • 不适用于 已持仓 减仓 决策
  • 不适用于 浮盈 > 30% 的票
```

### 🚨 漏洞 2：**缺少"换仓决策审查"机制**

```
当前流程:
  用户问操作 → 我答 操作
  
应该流程:
  用户问操作 → 我先审查:
    □ 这是 加仓 / 减仓 / 换仓?
    □ 这是 个股 / 板块 操作?
    □ 这是 计划内 / 计划外?
    □ 是否 同板块 同涨跌 风险?
    □ 是否 增加 集中度 风险?
    □ 用户 当前总仓位 / 现金 / 已实现盈利 是?
    □ 不操作 是否 也是 选项?
  → 然后 才 答
```

### 🚨 漏洞 3：**缺少 "板块相关性" 分析**

```
v4.1 缺失:
  • 同板块持仓识别
  • 板块集中度警示
```

### 🚨 漏洞 4：**缺少 "保护胜利果实" 模式**
