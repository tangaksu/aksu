# 我为什么开源了一个 A 股分析工具：从一笔失误说起

> 这个工具不是凭空来的，是踩了几次坑后磨出来的。

## 一笔本可避免的失误

2026 年 4 月某天早盘 9:25。我手里两只重仓股同时走弱：

- 股票 A：刚跌穿 MA20，浮盈 +42%
- 股票 B：放量回踩，浮盈 +18%

我打开券商 APP，习惯性问自己："要止损吗？"

接下来发生的事情很典型：
- 看了 5 分钟分时图，**心跳加速**
- 想想 +42% 的利润："再等等，这只票基本面还行"
- 又想想自己这周已经亏过钱："万一跌穿成本怎么办"
- 最后**什么都没做**，继续上班

中午吃饭再看 — 股票 A 又跌了 5%。

那天我意识到一件事：**散户最大的敌人不是市场，是自己的情绪。**

具体来说，是那 5 分钟里**信息处理混乱**：
- 没有事先想好的"触发条件"
- 没有事先算好的"具体动作"
- 在情绪状态下做技术判断 = 注定后悔

## 我想要的是什么

我想要在**每个交易日开盘前 5 分钟**，就拿到一张这样的清单：

```
| 优先级 | 股票     | 触发位 | 动作       |
|--------|----------|--------|------------|
| P0 🚨  | 股票A    | 487    | 减 1/3     |
| P1 🎯  | 股票B    | 96.10  | 减 1/2     |
```

然后**全部设成券商的条件单**，关掉 APP 去工作。盘中无论涨跌，都不打开。

这就是 stock-realtime-brief 想做的事。

## 它解决的三个具体问题

### 1. 数据源不稳定

A 股 Python 生态很丰富，但用户都知道 AKShare 主源 `82.push2.eastmoney.com` **在交易日早盘经常超时**。我自己测过最久 50+ 分钟没响应。

**解决**：默认走腾讯财经接口（不是腾讯股票，是 `qt.gtimg.cn`），它最稳。AKShare 仅用于历史 K 线（这接口稳定）。新浪批量作为备用。

### 2. "建议关注"是无效输出

Github 上很多 A 股分析工具的输出长这样：

> 该股近期走势偏弱，建议谨慎操作，注意风险。

这句话**没有触发条件、没有具体动作、没有数量**。换一句白话：等于没说。

**解决**：所有输出都遵循 SMART 原则。
- ❌ "建议谨慎" → ✅ "跌破 487.74（MA20）减 1/3"
- ❌ "情绪面不佳" → ✅ "成交 4.79 亿，量比 0.62（早盘缩量）"

### 3. 盈利股的止损线毫无意义

我见过一个常见反模式：

```python
def stop_loss(cost):
    return cost * 0.85   # 亏 15% 止损
```

成本 100，现价 150（盈利 50%）的票，这条规则给出的止损线是 85。
**距离现价 43%。这线设了等于没设。**

我的工具引入"利润保护位"：

```
当浮盈 > 30% 时:
profit_lock = 现价 - (现价 - 成本) × 25%

例：成本 100 / 现价 150
profit_lock = 137.5
```

**含义**："最多让出 25% 已实现利润"。这才是有意义的止损。

## 三种使用模式

工具自动根据你的输入判断要做什么：

```bash
# 你给一个代码 → 单股深度分析
$ stock-brief --code 300750

# 你给多个代码 → 横向对比 + 评分
$ stock-brief --codes 600519,300750,600036

# 你给一个 portfolio 文件 → 完整持仓简报（含融资风险 + 公告检测）
$ stock-brief --portfolio my_portfolio.json
```

最后一种模式（持仓简报）是最有用的。它会做几件别的工具不做的事：

1. **自动算担保比例**（如果你有融资融券）
2. **自动拉近 14 天重要公告**（减持/立案/业绩预减/解禁）
3. **按风险排序持仓**（亏损×融资×破位×重仓）
4. **生成可执行条件单清单**，直接抄进券商 APP

## 一个真实例子

5 月 6 日早盘，我跑了次持仓简报。工具输出告诉我：

> 📢 近 14 日重要公告告警
> - 🔴 **天岳先进** [5 days ago] [减持股份] 股东及其一致行动人减持股份计划公告

如果**没有这条提醒**，我可能就只看技术面，错过这个核心利空。

那天我借早盘强势卖了 8000 股 @均价 110，**已实现盈利 ¥14.62 万**。
接下来一个月不管股价怎么走，这 14.62 万是**确定性的**。

工具最大的价值不是"它判断得多准"，是它**强迫你按规则执行**。

## 设计哲学：规则比感觉重要

工具内置了一些"硬纪律"：

1. **三档止损**（预警/风控/清仓）— 不是一刀切
2. **重仓股容错收紧 5%** — 仓位越大，规则越严
3. **融资股止损位上提 3%** — 融资有杠杆，不能死扛
4. **亏损股 + 重仓 = 优先级 P0** — 风险排序自动化
5. **所有输出都带触发条件** — 杜绝"建议关注"

这些规则不一定每条都对。但**有规则总比没规则强**。
任何愿意按规则执行的人，长期一定跑赢"凭感觉"。

## 技术栈

- Python 3.10+
- akshare（历史数据）
- 自己写的腾讯/新浪适配（实时行情）
- gsk web_search（公告检测）
- pandas, requests（数据处理）

模块化设计，每个文件 ~150 行，方便贡献者切入。

## 它不能做什么

诚实声明：

- ❌ **不能选股**（这个工具是分析现有持仓/给定标的，不是发掘新机会）
- ❌ **不预测涨跌**（它给的是规则化决策框架，不是"明天涨还是跌"）
- ❌ **不构成投资建议**（详见 DISCLAIMER.md）
- ❌ **不支持港美股 / 期货 / 加密货币**
